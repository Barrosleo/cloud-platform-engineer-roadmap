# Week 2 — Linux Services, Logs & Networking  
## Linux Troubleshooting Lab

This lab demonstrates real operational troubleshooting tasks performed by Cloud, Platform, and Automation Engineers. The focus is on diagnosing failures using processes, logs, networking, and DNS, then documenting findings using a structured engineering workflow.

All work was performed using WSL2 with Ubuntu 22.04, reflecting how Linux is commonly accessed in modern cloud environments.

---

## What I Learned

- Process inspection and control using standard Linux tools.
- Log analysis across syslog, auth.log, and journalctl.
- DNS, routing, and name resolution troubleshooting.
- Network diagnostics using ping, curl, ip, and traceroute.
- Evidence‑based troubleshooting workflows.
- Clear documentation of root causes and validated fixes.

---

## Why Logs Matter

Logs provide the ground truth during incidents. They reveal:

- What failed  
- When it failed  
- Which component caused the failure  
- Whether the issue is recurring  

Effective log analysis reduces downtime and prevents guesswork.

---

## Why Networking Matters

Most cloud outages are caused by networking issues. Understanding interfaces, routing, ports, and DNS enables engineers to:

- Isolate issues quickly  
- Diagnose failures accurately  
- Restore service safely  

---

## Troubleshooting Workflow Used

1. Observe symptoms  
2. Collect evidence (logs, commands, outputs)  
3. Identify root cause  
4. Apply fix  
5. Verify recovery  
6. Document findings
